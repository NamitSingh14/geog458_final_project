About the project

Through this project, we will attempt to visualize minimum and maximum temperatures across the counties in the United States, along with the precipitation rate. Since the idea is to showcase climate change, the project will have the temperature and precipitation readings from different years. This project would be intended for the general public, specifically those who have an interest in the chosen location (the U.S). This could also benefit climate change activists and other people trying to study climate change in any location in the USA.

Data Source

The data was aggregated by the National Center for Environmental Information. They have hosted the weather data at https://www.ncei.noaa.gov/pub/data/cirs/climdiv. In this repository, they have provided the required shape files for contiguous USA and Alaska separately. Along with those shape files, we have a lot of exhaustive weather related data but we are only using the max, min and average temperatures, and precipitation data from 1895 to 2022. Each file contains county based data for each month from 1895 to 2022.

How to Interpret the Original Data

At first glance, the weather data files look very overwhelming and confusing to read. This is why NCEIâ€™s repository contains a divisional-readme.txt file at https://www.ncei.noaa.gov/pub/data/cirs/climdiv/divisional-readme.txt .

To simplify what the readme file contains, the following are the general rules each follows:

  1. All columns besides the first column are the respective weather readings for each consecutive month. For example: In the climdiv-tmaxdv-v1.0.0-20220406 file, the second column is the max temperature reading for January, third column is for February and so on until December which is the thirteenth column>
  2. The first column values are interpreted as follows:
    • First 4 (1-4) digits are the FIPS codes for counties
    • Next 2 (5-6) digits are the element code
    • Last 4 (7-11) digits is the year
    • 0101271895: 0101 is the FIP code, 27 is the element code, 1895 is the year

Element Codes:

  • 01 = Precipitation
  • 02 = Average Temperature
  • 27 = Maximum Temperature
  • 28 = Minimum Temperature
Note: These are not all the element codes. Only the ones we are concerned with are provided

Preprocessing

All the data preprocessing was done in MS Excel. This could be done in Python using its Pandas module but the required preprocessing was very simple so, MS Excel was sufficient. Since the first column contains information about the year and county FIPS code, we needed to split up the digits into separate columns. MS Excel has a function `MID` which lets you extract a sub-string from a given string. Using that, the first column was split into `FIPS_CD` and `Years` column for all files. Then columns for the months were created by entering `January` for the third column header and letting MS Excel autofill the rest of the months. The provided shape files contain the county FIPS code and the column is named `FIPS_CD`. This is why in the processed weather data files, the county FIPS code column was also named `FIPS_CD`.
